# Когортный анализ клиентов аптечной сети
> **Дисклеймер:** проект выполнен на синтетических данных выдуманной компании в образовательных целях для демонстрации аналитических навыков.

## 1. Контекст и постановка задачи

Сеть «Тест-Аптека» заинтересована в повышении долгосрочной ценности клиентской базы и эффективности программ удержания покупателей.

Цель исследования — провести когортный анализ клиентской базы с расчётом LTV, выявить различия в поведении клиентских когорт, определить закономерности формирования долгосрочной выручки и оценить динамику повторных покупок. На основе полученных результатов необходимо разработать рекомендации по повышению удержания клиентов, развитию программ лояльности и увеличению долгосрочной ценности клиентской базы.

## 2. Подготовка к проведению когортного анализа

### Подготовка данных для когортного анализа

``` sql
/*==================================================
  VIEW: checks_filtered
  Описание: Подготовка к проведению когортного анализа
==================================================*/

create or replace view checks_filtered as 
select
    card,
    datetime,
    summ_with_disc
from checks
where card like '2000%'
  and summ_with_disc > 0
  and datetime >= '2021-01-01'
  and datetime <= now()
order by datetime
```

Перед проведением когортного анализа была выполнена предварительная фильтрация данных для формирования однородной выборки клиентов и исключения записей, способных исказить результаты исследования.
Использовались следующие условия отбора:

* `datetime >= '2021-01-01'`

  В рамках поставленной бизнес-задачи анализируется поведение клиентов, привлечённых начиная с 2021 года. Ограничение периода позволяет сосредоточиться на актуальной клиентской базе и оценить эффективность удержания клиентов в рассматриваемом временном интервале.

* `datetime <= now()`

  Исключаются записи с датами, превышающими текущий момент времени. Данная проверка позволяет избежать влияния ошибок загрузки данных, некорректно указанных дат и других аномалий, способных исказить результаты анализа.

* `card LIKE '2000%'`

  В исследование включены только клиенты определённого сегмента программы лояльности, идентифицируемые по номеру карты, начинающемуся на «2000». Согласно условиям задачи данный сегмент представляет основной интерес для бизнеса и используется для оценки долгосрочной ценности клиентов.

* `summ_with_disc > 0`

  Исключены операции с нулевой или отрицательной суммой покупки. Для расчёта LTV учитываются только транзакции, формирующие выручку компании, поэтому в анализ включены исключительно фактические покупки клиентов.

### Анализ полноты данных по месячным когортам

``` sql
/*==================================================
  Описание: Анализ кол-ва дней в месяце
==================================================*/

select
    date_trunc('month', datetime)::date cohort_month,
    min(datetime)::date first_date,
    max(datetime)::date last_date,
    (max(datetime)::date - min(datetime)::date) + 1  days_in_month
from checks
where card like '2000%'
  and summ_with_disc > 0
  and datetime >= '2021-01-01'
  and datetime <= now()
group by date_trunc('month', datetime)::date
order by days_in_month
```

Анализ полноты данных показал, что июль 2021 и июнь 2022 содержат неполный набор дней. Для исключения искажения показателей LTV данные когорты были исключены из дальнейшего анализа.

## 3. Расчёт накопительного LTV по когортам

1. Для каждого клиента определяется когорта (месяц совершения первой покупки). Далее рассчитывается номер месяца жизни клиента (`num_of_months`) относительно месяца первой покупки.

2. Расчет среднего LTV выполняется как сумма покупок клиентов когорты за соответствующий период, деленная на количество клиентов в когорте.

3. Пустые значения означают, что когорта ещё не достигла соответствующего возраста на момент проведения анализа.

В итоговой таблице представлены значения накопительного LTV:

* `cohort` — месяц первой покупки клиента (когорта)
* `cohort_clients` — количество клиентов в когорте
* `m0` — средний LTV за первый месяц жизни клиента
* `m1` — средний накопительный LTV за первые два месяца жизни клиента
* `m2` — средний накопительный LTV за первые три месяца жизни клиента
* `m3` — средний накопительный LTV за первые четыре месяца жизни клиента
* и т.д

``` sql
/*==================================================
  Описание: Когортный анализ
==================================================*/

with pre_data as(
	select
		card,
		first_value(date_trunc('month', datetime)::date) over(partition by card order by date_trunc('month', datetime)::date) cohort,
		date_trunc('month', datetime)::date as month,
		summ_with_disc
	from checks_filtered
), pre_data_2 as(
	select
		card,
		month,
		cohort,
		(extract(year from month) - extract(year from cohort)) * 12 + 
		extract(month from month) - extract(month from cohort) num_of_months,
		summ_with_disc
	from pre_data
	where cohort not in ('2021-07-01', '2022-06-01')
)

select
	cohort,
	count(distinct card) as cohort_clients,
	round(sum(case when num_of_months <= 0 then summ_with_disc end) / count(distinct card)) m0,
	round(case when max(num_of_months) > 0 then sum(case when num_of_months <= 1 then summ_with_disc end) / count(distinct card) end) m1,
	round(case when max(num_of_months) > 1 then sum(case when num_of_months <= 2 then summ_with_disc end) / count(distinct card) end) m2,
	round(case when max(num_of_months) > 2 then sum(case when num_of_months <= 3 then summ_with_disc end) / count(distinct card) end) m3,
	round(case when max(num_of_months) > 3 then sum(case when num_of_months <= 4 then summ_with_disc end) / count(distinct card) end) m4,
	round(case when max(num_of_months) > 4 then sum(case when num_of_months <= 5 then summ_with_disc end) / count(distinct card) end) m5,
	round(case when max(num_of_months) > 5 then sum(case when num_of_months <= 6 then summ_with_disc end) / count(distinct card) end) m6,
	round(case when max(num_of_months) > 6 then sum(case when num_of_months <= 7 then summ_with_disc end) / count(distinct card) end) m7,
	round(case when max(num_of_months) > 7 then sum(case when num_of_months <= 8 then summ_with_disc end) / count(distinct card) end) m8,
	round(case when max(num_of_months) > 8 then sum(case when num_of_months <= 9 then summ_with_disc end) / count(distinct card) end) m9,
	round(case when max(num_of_months) > 9 then sum(case when num_of_months <= 10 then summ_with_disc end) / count(distinct card) end) m10
from pre_data_2
group by cohort
order by cohort
```

### Основные тренды по данным

- Основная часть накопления LTV формируется в первые **6–8 месяцев жизни клиента**. После этого периода темпы роста заметно снижаются, а вклад повторных покупок в общий LTV становится менее существенным.

- Наиболее **устойчивый рост LTV демонстрируют когорты августа–октября (2021) года**. Они показывают высокий темп накопления в первые месяцы жизни клиента и сохраняют хорошую динамику вплоть до M6. Возможно, это связано с сезонным спросом, более качественным привлечением клиентов или удачными маркетинговыми активностями.

- **Декабрьская (2021) и январская (2022) когорты выглядят менее устойчивыми** с точки зрения формирования LTV по сравнению с осенними когортами. Особенно выделяется декабрьская когорта, которая начинает отставать уже в первые месяцы жизни клиента.

- **Когорта марта (2022) года имеет самый высокий LTV** первого месяца, однако её дальнейший рост быстро замедляется, что является хорошим примером того, что **высокий чек первой покупки ещё не гарантирует высокий долгосрочный LTV.**

- **Когорты апреля и мая (2022)** года показывают **слабый прирост LTV** практически сразу после привлечения, что может указывать на снижение активности повторных покупок.

- Интересно выглядит **когорта августа (2021) года:** после **M6** она продолжает активно накапливать LTV, тогда как многие другие когорты уже выходят на плато. Это может говорить о **более высокой лояльности клиентов** данной когорты или особенностях их покупательского поведения.

### Ключевой вывод анализа

Результаты анализа указывают на существенные различия в поведении и долгосрочной ценности клиентов, привлечённых в разные периоды времени. Несмотря на схожие условия работы аптечной сети, отдельные когорты демонстрируют заметно разные темпы накопления LTV, что может свидетельствовать о различиях в качестве привлекаемых клиентов, эффективности удержания или влиянии внешних факторов. Для подтверждения причин выявленных различий требуется дополнительный анализ retention, частоты покупок и маркетинговых активностей.
